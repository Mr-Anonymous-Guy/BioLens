2.1  NEW USER REGISTRATION
─────────────────────────────────────────────────────────────────────────────

  Landing Page
        ↓
  Sign Up
        ↓
  ┌─────────────────────────────────────────────┐
  │  Pre-submit actions                         │
  ├─────────────────────────────────────────────┤
  │  Read terms      →  Terms of Service        │
  │  Read policy     →  Privacy Policy          │
  │  Consent toggle  →  Data Consent            │
  └─────────────────────────────────────────────┘
        ↓ (submit)
  Email Verification
        ↓
  ┌─────────────────────────────────────────────┐
  │  Email link clicked  →  Account Created     │
  │  Link expired        →  Resend verification │
  │                          ↓                  │
  │                       Email Verification    │
  └─────────────────────────────────────────────┘
        ↓ (verified)
  Account Created / Welcome
        ↓
  Onboarding Step 1 — Basic Profile


2.2  RETURNING USER LOGIN
─────────────────────────────────────────────────────────────────────────────

  Landing Page
        ↓
  Login
        ↓
  ┌─────────────────────────────────────────────┐
  │  Credentials valid     →  Main Dashboard    │
  │  Credentials invalid   →  Login (error)     │
  │  Forgot password       →  Forgot Password   │
  │  No account            →  Sign Up           │
  └─────────────────────────────────────────────┘


2.3  PASSWORD RECOVERY
─────────────────────────────────────────────────────────────────────────────

  Login
        ↓
  Forgot Password
        ↓
  [ Reset email sent to user ]
        ↓
  Reset Password                ← user clicks email link
        ↓
  ┌─────────────────────────────────────────────┐
  │  Reset success   →  Login                   │
  │  Token expired   →  Forgot Password (retry) │
  └─────────────────────────────────────────────┘
        ↓ (success)
  Login
        ↓
  Main Dashboard

        ↓
  Logout Confirmation
        ↓

  │               ↓                             │
  │            Landing Page                     │
  │  Cancel    →  Settings Hub                  │

─────────────────────────────────────────────────────────────────────────────

  Main Dashboard  /  Lab Test Results  /  Medical Reports
        ↓

        ↓
  Report Processing               ← OCR + biomarker extraction (polling)
        ↓
  ┌─────────────────────────────────────────────┐
  │  OCR success   →  Upload Success            │
  │  OCR failure   →  500 Error Page            │
  │                   ↓                         │
  │                Upload Medical Report (retry)│
  └─────────────────────────────────────────────┘
        ↓ (success)
  Upload Success
        ↓
  ┌─────────────────────────────────────────────┐
  │  View results   →  Lab Test Results         │
  │  View insights  →  AI Health Insights       │
  │  View reports   →  Medical Reports          │

        │
        └──→  [ connect ]

                  ↓
              Onboarding Step 4 (confirmed)   [ progress 4 / 6 ]
                  ↓
        Onboarding Summary                    [ progress 5 / 6 ]
        │
        ├──→  [ edit section ]  →  respective onboarding step
        │                              ↓
        │                         Onboarding Summary (return)
        │
        └──→  [ confirm ]

 SECTION 13 — CONDITIONAL FLOWS & GUARD RULES
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

  AUTHENTICATION GUARD
  ─────────────────────────────────────────────────────────────────────────
  Any protected page accessed without session
        ↓
  Redirect  →  Login
        ↓
  Post-login redirect  →  originally requested page


  ONBOARDING GUARD
  ─────────────────────────────────────────────────────────────────────────
  Authenticated user with incomplete onboarding accesses Dashboard
        ↓
  Redirect  →  last incomplete onboarding step


  EMAIL VERIFICATION GUARD
  ─────────────────────────────────────────────────────────────────────────
  Login attempt with unverified email
        ↓
  Redirect  →  Email Verification
        ↓
  Resend option available


  DEVICE CONNECTION (OPTIONAL STEP)
  ─────────────────────────────────────────────────────────────────────────
  Onboarding Step 4 — Device Connection
        ↓
  ┌──────────────────────────────────────────┐
  │  Skip      →  Onboarding Summary         │
  │  Connect   →  OAuth / Health API         │
  │               ↓                          │
  │            Success  →  Onboarding Summ.  │
  │            Fail     →  Step 4 (retry)    │
  └──────────────────────────────────────────┘


  EMERGENCY ALERT ESCALATION
  ─────────────────────────────────────────────────────────────────────────
  Anomaly detection fires critical severity alert
        ↓
  Push notification dispatched
        ↓
  User taps  →  Emergency Alert (full-screen)
        ↓
  ┌──────────────────────────────────────────┐
  │  Call emergency    →  tel: (external)    │
  │  Book doctor       →  Book Consultation  │
  │  Dismiss           →  confirmation modal │
  │                       ↓                  │
  │                    Main Dashboard        │
  └──────────────────────────────────────────┘

  │  User cancels      →  Lab Test Results   │

  ─────────────────────────────────────────────────────────────────────────

        ↓
